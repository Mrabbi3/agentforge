# AgentForge CI Integration

AgentForge can automatically analyze failing PRs and have an AI agent attempt to diagnose and fix issues. When a PR's CI tests fail, AgentForge kicks in, runs the agent, and posts results as a PR comment.

## How It Works

```
PR opened/updated
       |
       v
   CI runs tests ──── Pass ──> Done
       |
     Fail
       |
       v
  AgentForge triggers
       |
       v
  ci_analyze.py detects failures
       |
       v
  ci_agent_run.py runs the agent
       |
       v
  Posts results as PR comment
```

1. **CI workflow** runs your standard tests and lint checks
2. If CI **fails**, the `agent-fix.yml` workflow triggers automatically
3. `ci_analyze.py` inspects the PR diff, runs tests locally, and identifies what's broken
4. `ci_agent_run.py` creates task definitions and runs the AgentForge agent to analyze/fix issues
5. Results are posted as a **formatted PR comment** with status, metrics, reasoning, and tool usage

## Setup

### 1. Add your Anthropic API key as a GitHub secret

Go to your repo's **Settings > Secrets and variables > Actions** and add:

- **Name:** `ANTHROPIC_API_KEY`
- **Value:** Your Anthropic API key (starts with `sk-ant-...`)

### 2. Ensure workflows have PR permissions

The `agent-fix.yml` workflow needs `pull-requests: write` permission to post comments. This is already configured in the workflow file, but if your repository has restricted permissions, go to **Settings > Actions > General > Workflow permissions** and enable "Read and write permissions".

### 3. Push the workflow files

Make sure these files are in your repository:

```
.github/workflows/
  ci.yml              # Your standard CI workflow
  agent-fix.yml       # AgentForge auto-fix workflow

scripts/
  ci_analyze.py       # Failure detection script
  ci_agent_run.py     # Agent runner + comment generator
```

## Manual Trigger

You can also run the agent manually on any PR:

1. Go to **Actions > AgentForge Auto-Fix**
2. Click **Run workflow**
3. Enter the PR number
4. Click **Run**

## PR Comment Format

When the agent runs, it posts a comment like this:


## Configuration

The agent uses `configs/default.yaml` by default. You can customize:

- **Memory strategy**: sliding_window, summarization, hybrid
- **Max steps**: How many steps the agent can take per task
- **Model**: Which Claude model to use
- **Evaluation metrics**: Which metrics to compute

## Cost Considerations

Each agent run costs approximately $0.05-0.15 per task depending on complexity. For a typical PR with 1-2 failing tests, expect ~$0.10-0.30 per run. You can set spending limits in your Anthropic dashboard.

## Troubleshooting

**Agent doesn't trigger:** Check that the CI workflow name in `agent-fix.yml` matches exactly (case-sensitive): `workflows: ["CI"]`

**No PR comment appears:** Verify the `GITHUB_TOKEN` has `pull-requests: write` permission and the workflow isn't being blocked by branch protection rules.

**Agent can't fix the issue:** The agent works best on isolated bugs with clear test failures. Complex architectural issues or multi-file refactors may be beyond its current capability.
